# 4.2 Troubleshooting Process & PC Common Problems — Study Notes

## The 6-Step Troubleshooting Process

---

### Step 1: Identify the Problem
- Gather information about the issue from the user
- Observe symptoms and reproduce the problem if possible
- Ask open-ended and closed-ended questions

---

### Step 2: Establish a Theory of Probable Cause
- List possible causes, **starting with the most obvious and simplest**
- Prioritize: easy/common causes at the top, complex ones at the bottom
- Use internal resources (logs) or external resources (internet) if needed
- This sets up the next step — testing

---

### Step 3: Test the Theory to Determine the Cause
- Test each theory **one at a time**, starting with the quickest and easiest
- Testing one at a time ensures you know exactly what fixed the problem
- **Document every test and its result** in a Work Order
- If no theory proves correct → go back to Step 2 with new theories
- If escalating to another technician, documentation helps them work faster

---

### Step 4: Establish a Plan of Action & Implement the Solution
- Once the cause is confirmed, create and apply a fix
- Break large problems into smaller parts; solve **simplest first**
- If a fix doesn't work → **reverse it**, then try the next option
- Research resources if needed:
  - Helpdesk logs, manufacturer FAQs, technical websites, manuals, forums
- Always consider **company policies and the impact of changes** before acting

---

### Step 5: Verify Full Functionality & Implement Preventive Measures
- After the fix, verify the **entire system** works — not just the repaired part
- Make sure no **new problems** were introduced during repair
- Have the customer verify the fix themselves whenever possible
- **Verification checklist:**
  - Reboot the computer
  - Test multiple applications
  - Check network and internet connectivity
  - Print a test document
  - Confirm all peripherals work
  - Confirm no error messages appear

---

### Step 6: Document Findings, Actions, and Outcomes
- Explain the problem and solution to the customer verbally and in writing
- Let the customer test and confirm the resolution
- Record the following in your Work Order and technician's journal:
  - Description of the problem
  - Steps taken to resolve it
  - Components used
  - Total time spent
- Finalize and hand over all relevant paperwork to the customer

---

## 4.2.2.1 PC Common Problems & Solutions

Computer problems generally stem from **hardware**, **software**, **network issues**, or a combination.

| Component | Common Causes |
|---|---|
| **Storage Device** | Loose/incorrect cables, wrong drive format, incorrect BIOS/jumper settings |
| **Motherboard & Internal Components** | Loose cables, failed components, incorrect drivers, corrupted updates |
| **Power Supply** | Faulty PSU, loose connections, insufficient wattage |
| **CPU & Memory** | Faulty installation, incorrect BIOS settings, poor cooling/ventilation, compatibility issues |
| **Display** | Incorrect settings, loose cable connections, corrupted or missing drivers |

---

## Key Takeaways

1. Always start with the **simplest, most obvious cause** before moving to complex ones.
2. Test theories **one at a time** — never multiple at once.
3. **Documentation is critical** at every stage.
4. If stuck, go back to Step 2 and form new theories.
5. Always verify the **full system**, not just the repaired component.
6. Consider **company policies** before implementing any changes.
